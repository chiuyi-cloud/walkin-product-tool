# 島內散步｜產品與報價工具（2B 業務）

幫 2B 業務依客戶需求，快速查產品、撈定價、做報價試算、產出提案草稿。

## 怎麼開啟
- 用瀏覽器開 `index.html`，或在此資料夾執行 `python3 -m http.server 4180` 後開 `http://localhost:4180`。
- 報價單暫存於瀏覽器（localStorage），適合單一業務的作業暫存。

## 功能（v0.1 原型）
1. **產品搜尋** — 1038 項產品（主行程＋成本元件），可用關鍵字、類型、分類、地區、適合人數篩選；卡片顯示定價、建議人數、不接待時段、主題/ESG、官網與素材連結。點「加入報價」帶入報價單。
2. **報價試算** — 疊加項目、改數量/單價、設人數與利潤加成 %，自動算成本合計、成本/人、建議售價、每人單價；可匯出 CSV。
3. **提案產生** — 依報價單自動生成提案草稿（Markdown），含推薦行程、報價明細、ESG 效益；可複製或下載 .md，貼進 Google Slides／Docs 微調。
4. **過去提案搜尋** — 來源為你們的「2B提案簡報檢索表」，共 **291 筆歷史提案、全部可一鍵開啟簡報**。可依關鍵字（客戶/地點/元件）、地區、時長、人數、活動目的篩選，卡片顯示客戶、客單價、用了哪些遊程元件、負責人與簡報連結。

## 檔案
- `index.html` / `app.js` — 介面與邏輯
- `products.js` — Zoho 產品快照（由 `build_snapshot.py` 產生）
- `build_snapshot.py` — 把 Zoho 撈下來的原始結果整理成 `products.js`
- `proposals.js` — 過去提案快照（由 `build_proposals.py` 從 `_index.xlsx` 產生）
- `build_proposals.py` / `_index.xlsx` — 「2B提案簡報檢索表」下載檔與轉換腳本

## 資料來源與更新
- 產品資料為 **Zoho CRM Products 模組的快照**（point-in-time，共 1047 筆，清理後 1038 筆）。
- 定價邏輯：`產品-*`＝主行程（定價多為「每人區間」文字，需客製）；`元件-*`＝成本積木（有數字單價，如導覽 8000/場、保險、餐食、人力 1600）。
- 要更新快照：重新從 Zoho 撈 Products → 跑 `build_snapshot.py`。之後階段可改為**即時連 Zoho**（需後端 + OAuth，與訂單工具同一套基礎）。
