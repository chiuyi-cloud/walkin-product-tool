# 上線部署 + 報價/提案回存　規劃

目標：把「產品與報價工具」放到網路上（業務點網址就用），且業務新做的
- **報價** → 同時存成 **Google 試算表**（放指定資料夾）＋ 寫進 **Zoho 對應案子(Deal)**
- **提案** → 存成 **Google 文件/簡報**到「07-02 提案簡報」，並 **自動新增一列到「2B提案簡報檢索表」**（過去提案搜尋會自動出現新案）

---

## 一、架構（為什麼需要後端）

瀏覽器不能安全地保管 Google／Zoho 金鑰，所以要一個小後端代為寫入：

```
業務瀏覽器 (前端，靜態網頁)
      │  存報價 / 存提案
      ▼
後端 API (保管金鑰，代為寫入)
      ├──► Google Drive / Sheets / Docs   （建立試算表、文件、附加檢索表列）
      └──► Zoho CRM                         （更新/建立 Deal）
```

- 後端用 **Google 服務帳號(Service Account)** 寫 Drive/Sheets/Docs（不需每位業務各自登入）。
- 後端用 **Zoho OAuth(refresh token)** 寫 CRM。
- 前端沿用現有工具，加兩顆按鈕：「存報價」「存提案」。

---

## 二、你要準備的東西（這些只有你/管理員能做）

### A. Google（給後端寫 Drive 的權限）
1. 在 Google Cloud 建一個專案，啟用 **Google Drive API、Google Sheets API、Google Docs API**。
2. 建一個 **服務帳號(Service Account)**，下載它的 **JSON 金鑰**。
3. 把服務帳號的 email（長得像 `xxx@xxx.iam.gserviceaccount.com`）加為「編輯者」共用到：
   - 「07-02 提案簡報」資料夾（存提案用）
   - 一個「業務報價試算」資料夾（存報價用，可新建一個）
   - 「2B提案簡報檢索表」這份試算表（要附加新列）
   > 重點：服務帳號要能寫入，就必須被「共用」到上述資料夾/檔案。

### B. Zoho（給後端寫 CRM 的權限）
1. 到 Zoho API Console 建一個 **Self Client / Server-based 應用**，取得 Client ID / Secret。
2. 產生 **refresh token**（scope 需含 `ZohoCRM.modules.deals.ALL`）。
3. 決定報價要寫到 Deal 的哪些欄位（建議：`OrderAmount` 或 `Amount`，並在 Description/備註附報價明細與 Google 試算表連結）。

### C. 上線主機（擇一，需要你開帳號）
- 推薦 **Railway** 或 **Render**：可連 GitHub、push 後自動部署、有免費額度。
- 需要一個 GitHub 帳號放程式碼（也方便日後「改完 push 就自動更新」）。

### D. 存取控制（避免外人開到）
- 簡單版：整站一組共用帳密（HTTP Basic Auth，訂單工具後端已有同款機制可沿用）。
- 進階版：限公司 Google 帳號登入。

---

## 三、資料怎麼對應（我會照這個做）

**報價 → Google 試算表**：客戶、人數、利潤加成、逐項(項目/數量/單位/單價/小計)、成本合計、建議售價、每人單價、產生時間、業務。

**報價 → Zoho Deal**：依「客戶名稱」找對應 Deal（或讓業務在工具裡選 Deal）→ 更新金額欄位 + 在備註附上明細與 Google 試算表連結。

**提案 → Google 文件**：用現有「提案草稿」內容生成 Google 文件/簡報，存到 07-02。

**提案 → 檢索表新列**：依檢索表欄位（縣市/地點/時長/人數/活動目的/案例(超連結)/客單/元件…）自動填一列，案例欄連到剛建立的提案文件。

---

## 四、進行順序（建議）

1.（你）準備 A、B、C、D 的帳號與權限 → 把金鑰/ID 給我（或填進後端設定檔）。
2.（我）建後端 + 前端兩顆按鈕；先接 **Google 報價試算表**（最單純）跑通。
3.（我）接 **提案存 Drive + 登錄檢索表**。
4.（我）接 **Zoho Deal 回寫**（會先用測試案子確認欄位，不亂動正式資料）。
5.（你＋我）部署上線、設存取密碼、給業務網址。
