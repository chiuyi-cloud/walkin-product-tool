# 上線部署步驟（Railway，約 20 分鐘）

目標：把工具放上網路，業務用一個網址就能開、按「存報價」就把報價寫回 Drive。
平台用 **Railway**（連 GitHub、push 自動更新、有免費額度）。

> 你需要：一個 **GitHub** 帳號、一個 **Railway** 帳號（可用 GitHub 登入）。
> 金鑰**不會**進到程式碼，是貼在 Railway 的環境變數裡。

---

## 步驟 1：把程式放上 GitHub
我已在本機把專案初始化成 git 並做好第一次 commit（金鑰檔已被 .gitignore 排除，不會上傳）。你只要：

1. 到 https://github.com/new 建一個**私有 (Private)** 儲存庫，例如 `walkin-product-tool`（先別勾任何 README）。
2. 回到終端機，在專案資料夾執行（把 URL 換成你剛建立的）：
   ```
   cd "/Users/chiuyi/CODE/產品及報價工具"
   git remote add origin https://github.com/<你的帳號>/walkin-product-tool.git
   git branch -M main
   git push -u origin main
   ```
   （第一次 push 會要你登入 GitHub。）

> 不熟 git 也沒關係，這幾行跟我說一聲，我可以幫你跑。

## 步驟 2：在 Railway 建立服務
1. 到 https://railway.app/ → 用 GitHub 登入。
2. **New Project → Deploy from GitHub repo** → 選剛剛的 `walkin-product-tool`。
3. Railway 會自動偵測 Python、安裝 requirements.txt、用 Procfile 啟動。

## 步驟 3：設定環境變數（重點）
在該服務的 **Variables** 分頁，新增以下變數：

| 變數名稱 | 值 |
|---|---|
| `GOOGLE_SA_JSON_CONTENT` | 打開金鑰檔 `proposal-price-tool-….json`，把**整個內容**複製貼上（Railway 支援多行貼上） |
| `GDRIVE_QUOTE_FOLDER` | `1ZJV-U0VPInJdlFJwRsGGyvx8fLxMBDlV` |
| `AUTH_USER` | 自訂一組帳號，如 `walkin` |
| `AUTH_PASS` | 自訂一組較強的密碼（業務登入用） |

> 不用設 `PORT`，Railway 會自動給。

## 步驟 4：產生網址並開通
1. 服務的 **Settings → Networking → Generate Domain**，會得到一個網址，如
   `https://walkin-product-tool-production.up.railway.app`
2. 用瀏覽器打開 → 會跳出帳密（步驟 3 設的 `AUTH_USER`/`AUTH_PASS`）→ 登入。
3. 試一筆：搜尋產品 → 加入報價 → 按「💾 存報價」→ 看到「Google試算表：✓」→ 去 Drive 資料夾確認新檔。

## 步驟 5：把網址＋帳密給業務
業務只要打開網址、輸入帳密就能用，不用裝任何東西。

---

## 以後要改工具？
在本機改完 → `git push` → Railway **自動重新部署**（約 1–2 分鐘），線上版就更新了。
（這就是你之前問的：不是存檔就生效，而是 push 一次就自動上線。）

## 之後要補的（不影響先上線）
- Zoho 報價回寫、提案存 Drive＋登錄檢索表：設定好對應金鑰後，加環境變數即可啟用。
